## 🧭 웹사이트 개요

### 📌 프로젝트 소개
대전시티투어 웹사이트는 다양한 투어 상품과 정보를 제공하지만,  
기존 구조에서는 **정보 탐색의 불편함**과 **상호작용 부족**, **어긋난 디자인 시스템**이 존재했습니다.  
본 프로젝트는 이를 개선하여 **직관적인 탐색 구조 + 동적인 사용자 경험**을 제공하는 것을 목표로 리뉴얼되었습니다.

### 👤 대상 사용자
- 대전시티투어를 처음 이용하는 일반 사용자
- 투어 일정 및 예약 정보를 빠르게 확인하고 싶은 사용자
- 모바일 환경에서 정보를 탐색하는 관광객

### 🎯 사용 목적
- 투어 상품 정보 탐색 및 일정 확인
- 날짜 기반 투어 검색
- 예약 신청 유도
- 공지사항 및 이용 후기 확인

### ✨ 주요 특징
- 반응형 웹 기반 (PC / Tablet / Mobile)
- JavaScript 객체 기반 데이터 렌더링
- 스크롤, 클릭 중심의 인터랙션 강화
- 사용자 행동에 따른 UI 상태 제어

## 🧠 어려웠던 점 & 해결 방식

### 1️⃣ 반응형 구간별 인터랙션 제어
**문제**
- PC / Tablet / Mobile 구간별로 UI 구조와 동작이 달라 Swiper 슬라이드를 특정 해상도에서만 유지해야 했음

**해결**
- `resize` 이벤트와 break point 상수를 기준으로 미디어 쿼리 js 함수 작성
- Swiper 인스턴스를 `destroy()` / 재생성하는 구조로 분리
- 초기 실행과 resize 상황 모두 대응 가능한 패턴으로 정리

---

### 2️⃣ 날짜 기반 투어 검색 로직 설계
**문제**
- 투어별로 운영 월과 요일이 서로 다름
- 단순 조건문(switch)이 늘어나며 유지보수성이 급격히 저하됨

**해결**
- 투어 정보를 객체 배열(`productsObj`)로 구조화
- `month`, `day` 데이터를 기준으로 필터링 함수 분리
- 데이터 변경 시 로직 수정 없이 객체만 수정하도록 설계

---

### 3️⃣ 모달 간 Datepicker 상태 충돌
**문제**
- 한 투어의 예약 모달에서 선택한 날짜가
  다른 투어 모달에도 그대로 남아 UX 혼란 발생

**해결**
- 모달 오픈 시마다 Datepicker를 `destroy → 재생성`
- 현재 선택된 투어 정보를 기준으로
  선택 가능한 날짜만 활성화하도록 초기화 로직 구성

---

### 4️⃣ 공지사항 검색 + 페이지네이션 병행 처리
**문제**
- 검색 결과와 페이지네이션 로직이 서로 충돌
- 고정 게시글 처리까지 고려해야 하는 복합 조건 발생

**해결**
- 검색 결과를 별도의 배열로 관리
- 검색 모드 / 일반 모드를 분기 처리
- 기존 페이지네이션 로직을 최대한 유지하며 확장

## 🗂️ 데이터 구조

### 1️⃣ 투어 상품 정보  
📁 `js/sub01/object.js`

```js
productsObj = [
  {
    id: // 투어 상품 id
    filter: // 분류 필터 종류
    month: // 투어 진행 월
    day:  // 투어 진행 요일
    imgPath: // 상품 썸네일 이미지 경로
    label: // 투어 상품 분류
    title: // 투어 상품 이름
    schedule: // 일정 안내문구
    plan: {  // 투어 시간표
      icon: // 시간표 아이콘
      time: // 일정 시간
      content: // 일정 내용
    }
  }
]
```

### 2️⃣ 공지사항 데이터  
📁 `js/sub02/object.js`

```js
postObj = [
  {
    no:  // 공지 번호 또는 상단 고정 여부
    title: // 공지 제목
    writer: // 공지 작성자
    date: // 공지 작성 날짜
  }
]
```

### 3️⃣ 이용 후기 데이터  
📁 `js/sub03/object.js`

```js
reviewObj = [
  {
    id: // 리뷰 id
    title: // 리뷰 제목
    writer: // 리뷰 작성자
    date: // 리뷰 작성 날짜
    rating: // 좋아요 또는 개선이 필요해요
    content: // 리뷰 내용
    img01: // 이미지 파일 경로
    img02: // 이미지 파일 경로
    img03: // 이미지 파일 경로
  }
]
```
